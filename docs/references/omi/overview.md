# Omi como referencia

## Qué es Omi

`Omi` es un producto open source de asistente de escritorio y wearable con licencia MIT. Para NicoBot nos interesa sobre todo como referencia de experiencia de escritorio, no como base completa de producto.

Repositorio:

- <https://github.com/BasedHardware/omi>

## Por qué es una buena referencia

- Tiene una barra flotante y un panel conversacional muy pulidos.
- Resuelve bien el ritmo visual entre estado compacto y conversación expandida.
- Separa la experiencia del agente de otras capas de producto más complejas.
- Ya contiene patrones útiles para una app desktop centrada en presencia constante.

## Qué valor nos aporta en esta etapa

- referencia visual para el display del agente
- referencia de interacción para la barra flotante
- ejemplo de cómo mostrar thinking, respuesta y follow-up sin sobrecargar la UI

## Licencia

Omi usa licencia MIT. Eso permite copiar, adaptar y redistribuir partes del software, siempre que mantengamos el aviso de copyright y licencia correspondiente cuando reutilicemos código.

## Límite de uso en NicoBot

No vamos a importar Omi entero. En esta fase se toma como inspiración fuerte para:

- floating bar
- panel del agente
- greeting y follow-up dentro del mismo espacio conversacional

Queda fuera de esta adopción inicial:

- backend cloud
- auth
- servicios remotos
- infraestructura operativa ajena a la v1
